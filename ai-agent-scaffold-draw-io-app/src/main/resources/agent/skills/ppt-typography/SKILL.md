# PPT 排版与字体规范

## 字号体系

| 元素类型 | 字号范围 | 场景 |
|---------|---------|------|
| 封面主标题 | 40-48 | `title_slide` 的主标题 |
| 封面副标题 | 20-24 | 封面的副标题/日期 |
| 内容页标题 | 26-30 | `content_slide` 等页面的标题 |
| 正文要点 | 16-20 | 列表项、说明文字 |
| 大数字/KPI | 36-52 | `data_highlight` 的核心数字 |
| 卡片标题 | 18-22 | `card_3col` 的卡片标题 |
| 卡片描述 | 13-15 | `card_3col` 的卡片描述 |
| 表格内容 | 12-14 | `content_slide` 的表格 |
| 辅助信息 | 12-14 | 日期、来源、页码 |

## 留白规则

### 四边安全区
```
左安全区：x ≥ 0.8 英寸
右安全区：x + w ≤ 12.5 英寸
上安全区：y ≥ 0.2 英寸（头栏内）
下安全区：y + h ≤ 7.1 英寸（底栏上方）
```

### 元素间距
- 标题与正文之间：至少 0.4 英寸
- 要点之间行间距：通过 `\n` 换行，前端自动处理
- 左右分栏间距：至少 0.3 英寸
- 元素之间绝对不能重叠

### 内容密度
- 每页要点数：3-5个（最佳4个）
- 每个要点文字：不超过2行
- 金句/大数字页：宁可留白，不要塞满
- **Less is More**：观众3秒内要能抓住核心信息

## 文字层级

一个页面最多3个层级：

```
层级1（最高）：页面标题 — 大号+加粗+白色
层级2（中等）：小标题/卡片标题/数字 — 中号+加粗
层级3（最低）：正文/描述 — 小号+常规
```

## 列表排版规范

### 无序列表
```
• 第一条要点
• 第二条要点
• 第三条要点
```
- 用 `•` 开头（Unicode U+2022），不用 `-` 或 `*`
- 每条一行，不要嵌套子列表
- 3-5条最佳，超过5条考虑分页

### 有序列表（流程/步骤）
```
1. 第一步
2. 第二步
3. 第三步
```
- 仅用于有序流程/步骤
- 数字后加英文句号

## 对齐规则

| 场景 | 推荐对齐 | 说明 |
|------|---------|------|
| 页面标题 | `left` | 左对齐更现代 |
| 封面标题 | `center` | 居中更有仪式感 |
| 正文列表 | `left` | 左对齐便于阅读 |
| 大数字 | `center` | 居中视觉冲击 |
| 卡片内容 | `center` | 卡片内容居中更平衡 |
| 表格 | `center` | 表格内容居中 |

## 字体选择

- 所有文字统一使用 `Microsoft YaHei`（微软雅黑）
- 通过 fontSize 和 bold 区分层级，不要换字体
- 中文内容不要用英文衬线字体

## 常见错误

1. ❌ 封面标题字号 < 40 —— 太小，没有视觉冲击力
2. ❌ 正文字号 > 20 —— 太大，一页放不了多少内容
3. ❌ 要点超过 6 条 —— 信息过载，观众记不住
4. ❌ 大段连续文字 —— PPT不是文档，必须精简
5. ❌ 所有文字都加粗 —— 没有层级，等于没有重点
6. ❌ 同一页面超过3种字号 —— 视觉混乱
